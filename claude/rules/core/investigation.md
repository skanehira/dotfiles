# 調査作業のルール

- **網羅的に調べ尽くす**: 呼び出し元・呼び出し先の両方向、同名シンボルのシャドーイング / オーバーライド、設定ファイル・環境変数・ビルドスクリプト等の間接的な影響源まで確認する。「おそらくこうだろう」で打ち切らず、実際のコードで裏付けを取る
- **原因が見つかっても止めない**: 処理の流れを最後まで追って全体を理解し、多段の原因（根本原因 → 中間原因 → 直接原因）を見破る。データの変換・伝搬経路を端から端まで追跡する
- **報告**: 処理フロー全体のどの段階で問題が起きているかを明確にし、根本原因と表面的な症状を区別する。関連する他の潜在的問題も併せて報告する
