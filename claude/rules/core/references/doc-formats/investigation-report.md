# 調査結果レポートの構造規範

/utility-doc-audit のフォーマット適合チェックの正本。各項目は yes/no + 根拠箇所で判定できる形にしてある。

調査の**進め方**は rules/core/investigation.md が正本。本ファイルはその**成果物（報告書）の型**を定める。

## 構造（違反は major/minor で報告）

1. 冒頭メタ情報: 作成日・種別・調査対象（何を明らかにするための調査か = 調査の問い）があるか
2. 結論先行: 冒頭に「調査の問いへの答え」があり、本文を読まずに結論が分かるか。確度（確定 / 高確度の推定 / 未確定）が付いているか
3. 問いと答えの対応: 調査の問いが複数なら列挙され、それぞれに答え（または「未解決」の明示）が対応しているか
4. 根本原因と表面症状の区別: 原因分析型の調査では、多段の原因（根本 → 中間 → 直接）が区別して書かれ、処理フロー全体のどの段階で問題が起きているか明確か
5. 証拠と出典: 各結論に根拠（ファイルパス:行 / コマンドと出力 / URL）が紐づいているか。事実と推測（「おそらく」）が区別してマークされているか
6. 調査範囲の開示: 何をどう調べたか（探索範囲・方法）に加え、**調べなかった範囲・打ち切った深さ**が明示されているか（網羅性の限界の開示）
7. 関連する潜在的問題: 本題以外に見つかった問題が本題と混ざらず別節で報告されているか
8. 鮮度: 事実に確認日があるか（調査結果は時点スナップショットであり陳腐化する）

## 可読性（違反は suggestion で報告）

- 結論 → 根拠 → 詳細の順。生ログ・長いコマンド出力は末尾の付録に隔離されている
- 時系列の作業ログ（「まず〜した、次に〜した」）ではなく、問い → 答えの構造に再構成されている
- 事実を表で列挙する場合、出典列がある
