## シート構成パターン (2 シート)

### シート 1「料金プラン」(営業向け・非エンジニア向け)

上から読めば完結する 1 枚構成。設定セルと収支が同じシートにあることが重要 (別シートだと非エンジニアが行き来できない):

1. タイトル + 「黄色のセルを編集すると自動再計算」の案内
2. **この料金の考え方** (平易な 3 行: 固定費は基本料で / 変動費は従量で / 繁忙期は収入も増える)
3. **料金表** (課金先 × 月額 × 単位 × 含まれるもの。月額は設定セル参照)
4. **前提と設定** (黄色 = 入力セル)。グループ: 【契約構成】【料金】【人件費】【インフラ】
5. **月次収支シミュレーション** (平均月 / 繁忙月の 2 列。収入内訳 → コスト内訳 → 利益・利益率)
6. **年間収支** (繁忙月数で加重)
7. **備考** (課金単位の定義、最小構成の黒字設計、原価の織り込みなど)

### シート 2「インフラ詳細」(エンジニア向け)

1. 使用量モデル (入力セル)
2. 認証サービスの費用計算
3. ストレージ収容設計 (ハード上限 × 使用率上限 → 収容テナント数 → 原価按分)
4. **感度分析の早見表** (ユーザー数別・テナント数別。黄色の入力列 + 行ごとの数式)
5. 無料枠判定 (使用量 vs 上限を IF で「枠内/超過」表示)
6. 月額インフラ費まとめ (料金プランシートが参照する集計セル)

## Numbers 互換ルール (実地検証済み・必読)

Numbers の xlsx インポータは取り込めない数式を**警告付きで値 0 に置換**する (openpyxl は計算結果キャッシュを書かないため 0 になる)。以下に違反すると発生する:

| ルール | NG 例 | OK 例 |
|--------|-------|-------|
| **シート間参照はクォートなし・相対参照** | `='料金プラン'!$B$19` | `=料金プラン!B19` |
| シート名に記号を使わない (クォート不要な名前に) | `設定（ここを編集）` | `設定` `料金プラン` |
| 「=」で始まるテキストを書かない (openpyxl が数式として保存) | 備考セルに `= 単価 × 時間` | `計算: 単価 × 時間` |
| 開いた時に全再計算させる | (未設定 → Excel でも 0 表示) | `wb.calculation.fullCalcOnLoad = True` |

補足:
- シート内参照は絶対参照 (`$B$25`) で問題ない。壊れるのは**シート間**のみ
- ROUNDDOWN / MAX / VLOOKUP / IFERROR は Numbers で動作確認済み
- 割り算セルは `IFERROR(x/y*100,0)` でガード (0 除算で赤エラー表示になる)
- 小額単価 ($0.015 等) は `'"$"#,##0.000'` (3 桁)。2 桁だと $0.02 に丸め表示され誤解を招く

## openpyxl の定型パターン

```python
SOFT_GOLD = "FEF3C7"   # 入力セル (黄色)
LIGHT_GRAY = "F3F4F6"  # 計算セル (グレー)
YEN_FMT = '#,##0"円"'
USD3_FMT = '"$"#,##0.000'  # 小額単価用

# シート間参照はクォートなしで組み立てる
PLAN_REF = PLAN_SHEET  # クォートしない (シート名に空白・記号を含めないこと)
formula = f"={PLAN_REF}!B19*(12-{PLAN_REF}!B21)"
```

- 入力セル (黄色) / 計算セル (グレー) を関数で統一し、ラベル・値・備考 (計算式の説明) を 1 行で置く
- 早見表: A 列を黄色の入力値 (50/100/150/...) にし、B〜E 列に行番号を使った数式を生成する
- 行番号のズレが最大の事故源。セクション追加時は**以降の全行番号と参照先をまとめて更新**し、必ず検証を通す

## 検証手順 (数式を修正したら毎回)

### 1. Numbers 実機読み取り (必須)

`assets/verify_numbers.sh` を使う。Numbers で実際に開いてセルの計算値を読み取る:

```bash
bash ~/.claude/skills/saas-pricing-design/assets/verify_numbers.sh \
  docs/料金プラン.xlsx "料金プラン:B50" "料金プラン:B55" "インフラ詳細:B22"
```

- 期待値は同じロジックを Python で並行計算して照合する
- `0.0` や `missing value` が出たら数式の取り込み失敗 → 上の互換ルール違反を疑う

### 2. 数式エンジンでの数学検証 (補助)

Numbers を開けない環境では `formulas` パッケージでブック全体を評価できる:

```bash
uv run --quiet --with formulas --with openpyxl python -c "
import formulas
xl = formulas.ExcelModel().loads('docs/料金プラン.xlsx').finish()
sol = xl.calculate()
# sol のキーから対象セルを検索して値を確認
"
```

注意: このエンジンは Excel 文法として正しければ通るため、**Numbers 互換の検証にはならない**。最終確認は必ず 1 の実機読み取りで行う。

## トラブルシューティング

| 症状 | 原因 | 対処 |
|------|------|------|
| Numbers で全セル 0 円 (青三角) | シート間参照のクォート/絶対参照、または「=」始まりテキスト | 互換ルール表を確認。最小プローブ (2 セルの xlsx) で切り分け |
| 利益率が赤三角 (エラー) | 0 除算 (収入 0 のカスケード) | IFERROR ガード + 根本原因の 0 を直す |
| Excel でも 0 表示 | fullCalcOnLoad 未設定 | `wb.calculation.fullCalcOnLoad = True` |
| 単価が $0.02 と表示される | 表示フォーマットの桁不足 | USD3_FMT (3 桁) を使う |
