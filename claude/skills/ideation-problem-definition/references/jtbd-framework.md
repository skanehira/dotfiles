## JTBDとは

**「ユーザーは製品を買うのではなく、ジョブを片付けるために製品を雇う」**

クレイトン・クリステンセンが提唱した理論。ユーザーの行動を「機能」ではなく「達成したい進歩」で理解する。

### ミルクシェイクの例

マクドナルドのミルクシェイク売上を伸ばすために調査した結果：

- 朝の通勤客が多く購入していた
- 彼らの「ジョブ」は「退屈な通勤時間を楽しく過ごす」「昼まで空腹を感じない」だった
- 競合はバナナ、ベーグル、ドーナツであり、他社のミルクシェイクではなかった

**製品カテゴリではなく、ジョブで競合を考える**。

## ジョブの構造

### 3層構造

| 層 | 説明 | 例（DBTune） |
|----|------|--------------|
| **機能的ジョブ** | 実用的に達成したいこと | クエリの実行時間を短縮したい |
| **感情的ジョブ** | どう感じたいか | 自信を持って対応したい、不安を解消したい |
| **社会的ジョブ** | 他者からどう見られたいか | チームから頼りにされたい |

### ジョブの4要素

1. **状況（When）**: どんな状況でジョブが発生するか
2. **動機（Why）**: なぜそれを達成したいか
3. **成果（What）**: 何を達成したいか
4. **制約（How）**: どんな制約の中で達成したいか

## ジョブステートメントの書き方

### 基本形式

```
[状況]のとき、
[ターゲットユーザー]は
[達成したいこと]したい。
なぜなら[動機/理由]だから。
```

### 良い例

```
スロークエリが発生したとき、
DBA不在のスタートアップの開発者は
本番環境に影響を与えずに改善案を検証したい。
なぜなら試行錯誤で本番を壊すリスクを取れないから。
```

### 悪い例

```
❌ 「ユーザーはDBを最適化したい」
   → 抽象的すぎる、状況がない

❌ 「ユーザーはDBTuneを使いたい」
   → 解決策を含んでいる（ジョブではない）

❌ 「クエリが遅いので速くしたい」
   → 動機がない、なぜ速くしたいか不明
```

## ペイン・ゲイン分析

### ペイン（痛み）の種類

| 種類 | 説明 | 質問例 |
|------|------|--------|
| **機能的ペイン** | うまく動かない、時間がかかる | 「どこに一番時間がかかる？」 |
| **金銭的ペイン** | コストがかかる | 「今いくらかかっている？」 |
| **プロセスペイン** | 手順が面倒、非効率 | 「一番面倒な手順は？」 |
| **サポートペイン** | 助けが得られない | 「困ったとき誰に聞く？」 |
| **感情的ペイン** | 不安、ストレス | 「どんなとき不安になる？」 |

### ペインの評価軸

- **深刻度**: 1-5（5が最も深刻）
- **頻度**: 毎日/週/月/四半期/年
- **代替策の有無**: 今どう対処しているか

### ゲイン（得たい価値）の種類

| 種類 | 説明 | 例 |
|------|------|-----|
| **必須ゲイン** | これがないと解決にならない | 改善効果が数値でわかる |
| **期待ゲイン** | あって当然と思う | 使いやすいUI |
| **望外ゲイン** | 期待を超える驚き | AIが自動で改善案を提案 |

## よくある間違い

### 1. 解決策をジョブと混同する

```
❌ 「ユーザーはダッシュボードが欲しい」
✅ 「ユーザーはシステムの状態を一目で把握したい」
```

### 2. 機能的ジョブだけに注目する

感情的・社会的ジョブを無視すると、機能は満たしても「愛される」製品にならない。

### 3. ペインを解決策で語る

```
❌ 「自動化ツールがない」（解決策の不在）
✅ 「手動で繰り返し同じ作業をしている」（ペインそのもの）
```

### 4. ターゲットを広く取りすぎる

```
❌ 「すべての開発者」
✅ 「DBA不在の10-50人規模のスタートアップで、週に1回以上スロークエリに直面する開発者」
```

## インタビュー質問テンプレート

### ジョブを特定する質問

1. 「最後に〇〇した時のことを教えてください」
2. 「その時、何を達成しようとしていましたか？」
3. 「なぜそれが重要だったのですか？」
4. 「うまくいきましたか？何が障害でしたか？」

### ペインを深掘りする質問

1. 「今のやり方で一番困っていることは？」
2. 「それによってどんな悪影響がありますか？」
3. 「どれくらいの頻度で起こりますか？」
4. 「今はどうやって対処していますか？」
5. 「それで満足していますか？」

### ゲインを特定する質問

1. 「理想的にはどうなっていたいですか？」
2. 「それが解決したら、次に何をしますか？」
3. 「期待を超えるとしたら、どんな体験ですか？」
