# AGENTS.md

## Repository Overview

This is a dotfiles repository managed with Nix.

- macOS uses nix-darwin plus Home Manager.
- Linux uses Home Manager standalone.
- Most editable configuration is linked with `mkOutOfStoreSymlink` so changes can take effect without copying files into the Nix store.

## Working With This Repo

- Use `rg` / `rg --files` for search.
- Do not use `sudo -n`; this machine uses Touch ID through PAM, and non-interactive sudo bypasses that prompt.
- For macOS activation, use `drs`.
- For Linux activation, use `hms`.
- New files referenced by the flake must be tracked by git before `drs` / `hms`, because flakes ignore untracked files.
- Do not revert unrelated user changes. In particular, treat existing local changes under `claude/` as user-owned unless asked otherwise.

## Codex Setup

- `codex/config.base.toml` is the tracked Codex config source.
- `~/.codex/config.toml` is generated by Home Manager activation and keeps local `[projects.*]` trust entries from the previous file.
- Do not add project trust entries, auth files, sqlite state, logs, history, or caches to git.
- `codex/AGENTS.md` is linked to `~/.codex/AGENTS.md` as global Codex guidance.
